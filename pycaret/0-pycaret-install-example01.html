Edimer David Jaramillo (Sidereus)¶

"PyCaret is an open-source, low-code machine learning library in Python that aims to reduce the cycle time from hypothesis to insights. It is well suited for seasoned data scientists who want to increase the productivity of their ML experiments by using PyCaret in their workflows or for citizen data scientists and those new to data science with little or no background in coding. PyCaret allows you to go from preparing your data to deploying your model within seconds using your choice of notebook environment". Homepage Pycaret

Etapa 1: configurar entorno de pycaret¶

  • La configuracin del entorno se lleva a cabo con la funcin setup(). Esta funcin requiere dos argumentos obligatorios:
    • data: base de datos con la cual se ajustan los modelos
    • target: variable respuesta
  • Es necesario que las variables estn tipificadas de la manera correcta, ya que pycaret realiza algunos procedimientos de preprocesamiento internamente.
  • Al ejecutar setup() con los argumentos necesarios, mostrar el tipo de cada variable, de ser correcta la tipificacin se presiona enter y obtendremos una descripcin detallada de los inputs y de los procedimientos de preprocesamiento aplicados, por ejemplo, el mtodo de imputacin es simple (por defecto). Este proceso es automtico, pycaret est diseado para hacer mucho con poco cdigo.
  • El argumento session_id acta como semilla para garantizar que los resultados sean replicables.
  • Internamente la funcin setup() asigna aproximadamente el 70% de los datos para train y el 30% restante lo usa para test.
  • El nmero de variables en la data original (Original Data) ha aumentado por la codificacin de variables categricas. Inicialmente la base de dates contiene 24 columnas y despus de aplicar setup() obtenemos 88 columnas.
  • Por defecto se ajusta como mtodo de validacin la validacin cruzada estratificada con $k = 10$ (Fold Number).

Finalizando ajuste de modelo¶

  • Todo proceso de modelamiento tpico podra ser resumido como se muestra en la siguiente imagen (tomada del libro Tidy Modeling With R - Max Kuhn y Julia Silge)

  • Inicialmente exploramos nuestros datos, se incorpora alguna estrategia de ingeniera de caractersticas, se comparan modelos, evaluamos el rendimiento de los mismos y, posiblemente surgen algunos candidatos sobre los cuales haremos un proceso ms exhaustivo de ajuste. Luego, terminamos de afinar nuestros modelos, usualmente con mayor rigor en la ingeniera de caractersticas, volvemos a evaluar nuestros algoritmos y, para terminar, finalizamos nuestro flujo de trabajo desplegando el o los mejores modelos. Este proceso lo podramos resumir en las siguientes etapas, haciendo nfasis en que para cada una de ellas pycaret suministra alguna funcin que reduce tiempo y lneas de cdigo:
    1. Depuracin y estructuracin de datos Esta es una fase obligatoria del modelado, aunque no es inherente al ajuste de modelos, por tal razn est fuera del alcance de pycaret y no se muestra en el esquema anterior. Bibliotecas como pandas en Python y dplyr en R, son herramientas flexibles y de gran utilidad para dar forma a nuestros datos.
    2. Anlisis exploratorio de datos Las herramientas de visualizacin como ggplot2 de R, plotly (R y Python) o matplotlib (Python), son buenas opciones para explorar nuestros datos.
    3. Preprocesamiento e ingeniera de caractersticas La funcin setup() permite de manera fcil el preprocesamiento y algunas estrategias de ingeniera de caractersticas.
    4. Comparacin de modelos La funcin compare_models() es una excelente herramienta que permite evaluar rapidamente cules modelos tienden a comportarse mejor para nuestro problema de prediccin. En esta fase tambin podramos utilizar la funcin create_model().
    5. Refinacin de modelos La funcin tune_model() permite *afinar" nuestros modelos en aras de encontrar la combinacin de hiperparmetros ptima.
    6. Modelo final la funcin finalize_model() permite ajustar el modelo con todos los datos, incluido el de validacin o test (interno). Con este modelo final podemos realizar predicciones sobre observaciones no vistas por el modelo.
